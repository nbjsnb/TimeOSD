# TimeOSD 功能说明（V1.0）

## 项目定位

TimeOSD 是一个面向视频后期打点的桌面工具，重点解决以下场景：

1. 运动相机缩时视频通常没有可用时间戳。
2. 运动相机增稳后，机内叠加标签无法稳定跟随画面。
3. 需要在后处理中批量给视频补充时间 OSD（日期/时间/帧号/真实经过时间）。

本工具支持 **缩时拍摄** 与 **正常拍摄** 两类视频，在导出时统一烧录 OSD。

---

## 关键能力

## 1. 面向运动相机无时间戳场景

- 支持读取视频元信息（时长、帧率、分辨率）。
- 支持以文件创建时间/手动时间作为起点时间。
- 支持将“拍摄间隔”映射到每一帧时间，补齐无时间戳素材的真实时间序列。

## 2. 缩时与正常速率双模式

- 缩时模式：按“拍摄间隔(秒)”进行时间推进。
- 正常速率模式（1秒=1秒）：自动按 `1/fps` 推进，无需手工输入 0.033333。
- 互斥策略：
  - `正常速率视频` 与 `ASS按帧更新` 互斥，避免配置歧义。

## 3. OSD 实时预览与交互

- 视频播放预览 + 可拖动进度条。
- 参数改动实时反映在左侧预览画面。
- 支持鼠标直接拖拽字幕位置（自动切换到 free 定位）。
- 支持模板字段：
  - `{date}` `{time}` `{datetime}` `{frame}` `{elapsed_real}`

## 4. 字幕生成与加载预览

- 生成 ASS 字幕文件（会覆盖同名文件）。
- 可加载 ASS 进行回放预览。
- 支持两种更新策略：
  - 固定频率更新（1~5次/秒）
  - 按帧更新（更精细，文件更大）

## 5. 导出渲染

- 导出 20 秒测试片（快速验证参数）。
- 导出全片。
- 编码器可选：
  - `h264_nvenc`（GPU）
  - `libx264`（CPU）

说明：字幕滤镜（ass）渲染通常由 CPU 参与，视频编码阶段可走 GPU（NVENC）。

## 6. 配置预设

- 保存当前配置
- 一键加载配置
- 删除配置
- 预设文件位于项目根目录：`timeosd_presets.json`

---

## 典型使用流程

1. 加载视频。
2. 设置时间模式（缩时/正常速率）。
3. 调整模板、样式、位置（可直接拖拽）。
4. 先导出 20 秒测试片检查效果。
5. 满意后导出全片。

## V1.0 范围

V1.0 目标是稳定完成“无时间戳视频补时 + OSD 烧录”完整闭环，适配运动相机后处理工作流。
